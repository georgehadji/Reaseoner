│  Implementation Plan                                                                                 │
│                                                                                                      │
│  Phase A: Source Retrieval Resilience (Files: core/search.py, article_pipeline.py)                   │
│                                                                                                      │
│  A1. Fix Perplexity result URL handling                                                              │
│                                                                                                      │
│  • In PerplexitySearchClient.search, when citations is empty, set url to "perplexity://synthetic" o  │
│    a synthetic marker instead of "".                                                                 │
│  • In _phase_article_retrieve, relax the if res.get("url") filter to accept Perplexity synthetic UR  │
│    .                                                                                                 │
│  • Add source_type: "synthetic" so downstream phases know this is a synthesized answer, not a prima  │
│    source.                                                                                           │
│                                                                                                      │
│  A2. Fix get_search_client to actually fallback                                                      │
│                                                                                                      │
│  • Change the logic so that when SearXNG is healthy (searxng_open == False), it tries SearXNG first  │
│    nd only falls back to Perplexity on failure.                                                      │
│  • When SearXNG is down (searxng_open == True), use Perplexity but log the reason.                   │
│  • If Perplexity raises ValueError (init failure) or returns empty results, fall back to SearXNG as  │
│    ast resort.                                                                                       │
│                                                                                                      │
│  A3. Add retrieval retry with relaxed filters                                                        │
│                                                                                                      │
│  • In _phase_article_retrieve, when ranked_sources is empty after the first pass, retry with:        │
│    • Broader queries (drop domain constraint, use state.problem only)                                │
│    • Relaxed quality filters (temporarily raise _MIN_SNIPPET_LEN to 20 or skip title-based filterin  │
│    • Different source_type (e.g., try "news" for current-events queries)                             │
│  • Log each retry attempt and the filter settings used.                                              │
│                                                                                                      │
│  A4. Improve retrieval diagnostics                                                                   │
│                                                                                                      │
│  • Log the raw result count before filtering, the number filtered out, and the reason (extension, l  │
│    gth, title pattern, domain blocklist).                                                            │
│  • Store retrieval diagnostics in state.writing_state["retrieval_diagnostics"] for UI visibility.    │
│                                                                                                      │
│  Phase B: Evidence Gate & Pipeline Integrity (Files: article_pipeline.py, pipeline.py, flows/__init  │
│  __.py)                                                                                              │
│                                                                                                      │
│  B1. Add minimum-evidence gate before synthesis                                                      │
│                                                                                                      │
│  • In _phase_article_synthesize, before falling back to monolithic synthesis, check:                 │
│    • len(state.writing_state.get("retrieved_sources", [])) >= ARTICLE_MIN_SOURCE_COUNT               │
│    • len(usable_claims) > 0                                                                          │
│  • If both fail, set state.writing_state["article"] to a structured "insufficient evidence" respons  │
│    instead of a synthetic article:                                                                   │
│    # Insufficient Evidence                                                                           │
│    No authoritative sources could be retrieved for this topic.                                       │
│                                                                                                      │
│    ## What was attempted                                                                             │
│  - <list search queries>                                                                             │
│  - <list fallback attempts>                                                                          │
│                                                                                                      │
│    ## Recommendations                                                                                │
│  - Try a more specific query                                                                         │
│  - Check search backend connectivity                                                                 │
│  • Set state.writing_state["insufficient_evidence"] = True so the UI can render a special state.     │
│  • Skip pre-mortem, critic, and humanize phases when insufficient evidence is detected.              │
│                                                                                                      │
│  B2. Implement critical phase halting                                                                │
│                                                                                                      │
│  • In PipelineFlow.execute_sequence (new method) or directly in ReasonerPipeline.run, wrap each ste  │
│    for step in sequence:                                                                             │
│      await step.fn(state)                                                                            │
│      if step.critical and _phase_failed(state, step.name):                                           │
│          state.errors.append(f"Critical phase '{step.name}' failed — halting pipeline.")             │
│          break                                                                                       │
│  • Define _phase_failed based on phase-specific failure indicators (e.g., for Adversarial Verify: m  │
│    rics.claim_support_ratio < ARTICLE_MIN_CLAIM_SUPPORT_RATIO and re-retrieval already attempted).   │
│  • For the writing flow, mark "Retrieve Sources" and "Adversarial Verify" as critical (already done  │
│    or Verify; add critical to Retrieve).                                                             │
│                                                                                                      │
│  B3. Fix Deep Read for writing/research flows                                                        │
│                                                                                                      │
│  • In ReasonerPipeline.run, run _phase_deep_read for the writing method when state.task_type is ANA  │
│    TICAL or HYBRID, regardless of edu/religious keyword heuristics.                                  │
│  • OR: merge Deep Read results into the article retrieval phase so vetted context is always availab  │
│    as a fallback.                                                                                    │
│                                                                                                      │
│  Phase C: Pre-Mortem, Abstract Deduplication, Metrics (Files: article_pipeline.py, api/serializers.  │
│  py)                                                                                                 │
│                                                                                                      │
│  C1. Fix pre-mortem serializer to show revised article                                               │
│                                                                                                      │
│  • In _ser_writing_premortem, prefer writing.get("article_revised") over writing.get("article").     │
│  • If article_revised exists, show both original and revised with a diff indicator.                  │
│                                                                                                      │
│  C2. Fix pre-mortem correction application                                                           │
│                                                                                                      │
│  • In _phase_article_pre_mortem, after _apply_corrections_to_article, verify the article actually c  │
│    nged by comparing lengths or content hash.                                                        │
│  • If no change occurred, log a warning and try a stronger revision prompt that explicitly requires  │
│    ewriting flagged sections.                                                                        │
│                                                                                                      │
│  C3. Deduplicate abstract in assembly                                                                │
│                                                                                                      │
│  • In _phase_article_assemble, before adding the abstract field, check if any section heading (case  │
│    nsensitive) matches "abstract", "summary", or "overview".                                         │
│  • If found, skip adding the separate abstract and log: "Abstract already present in sections — ski  │
│    ing duplicate."                                                                                   │
│  • Also deduplicate in _phase_article_synthesize_monolithic: if the reconstructed article text alre  │
│    y contains an abstract heading, don't prepend another.                                            │
│                                                                                                      │
│  C4. Fix quality metrics for zero-claim state                                                        │
│                                                                                                      │
│  • In _calculate_metrics, when total == 0, return a sentinel state instead of all zeros:             │
│    return ArticleMetrics(                                                                            │
│      total_claims=0,                                                                                 │
│      verified_claims=0,                                                                              │
│      weak_claims=0,                                                                                  │
│      conflicted_claims=0,                                                                            │
│      unknown_claims=0,                                                                               │
│      citation_accuracy=-1.0,  # sentinel: not applicable                                             │
│      contradiction_rate=-1.0,                                                                        │
│      claim_support_ratio=-1.0,                                                                       │
│    )                                                                                                 │
│  • In _phase_article_assemble, when rendering metrics, check for the sentinel and display "N/A (no   │
│    aims tracked)" instead of 0.0.                                                                    │
│  • Update the frontend SynthesisCard or PhaseRenderer to handle negative sentinel values.            │
│                                                                                                      │
│  Phase D: Timeout & Model Diversity (Files: article_pipeline.py, pipeline.py)                        │
│                                                                                                      │
│  D1. Add timeout wrapper to Journal Review                                                           │
│                                                                                                      │
│  • Wrap _phase_article_critic in an asyncio.wait_for with a configurable timeout (default 60s).      │
│  • On timeout, log the timeout, set critic_score = 0, must_revise = False, and append a warning eve  │
│    . Do not block the pipeline.                                                                      │
│                                                                                                      │
│  D2. Enforce cross-model diversity in article phases                                                 │
│                                                                                                      │
│  • In preset_registry.py, add explicit routing entries for article-specific roles:                   │
│    • article_synthesize: different from article_pre_mortem and article_critic                        │
│    • article_pre_mortem: use a destructive/critical model                                            │
│    • article_critic: use a scoring/verifier model                                                    │
│  • Update research-budget and research-premium presets with these mappings.                          │
│  • In _PHASE_CONFIGS in pipeline.py, ensure article roles have distinct model assignments.           │
